Lab → Salesly Bridge

Upload the daily lab CSV once. This classifies every practice (Active / New Doctor / Winback / Falloff / Inactive), geocodes any office Salesly hasn't seen before, and pushes it all into the reps' app through the same Google Sheet it already syncs with. Reps see the update within ~15 seconds.

1 Connect (saved on this device)

Use the exact same Apps Script URL and sync token as the Salesly app's Team Sync screen. The ORS key is optional but makes geocoding ~3× faster — get a free one at openrouteservice.org (the free tier covers ~1,000 lookups/day; after the first big run you'll only geocode a handful of new offices per day).

2 Upload the daily lab CSV

The same file you feed the dashboard (case-level export with pracname, address, city, state, Invoice_Date, Net_Cost…). Large files are fine — it's all processed in your browser; nothing is uploaded anywhere except the final result to your own Google Sheet.
Status rules (match these to your dashboard tabs)
"Today" is the newest invoice date in the file, so re-running an old file gives the same answer. Priority when several apply: New Doctor → Winback → Active → Falloff → Inactive.

4 Sync to the reps' app
